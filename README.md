## 📂 Directory Map (目录映射)

This project treats religions as software repositories. Choose your stack:
(本项目将宗教视为软件仓库。请选择你的技术栈：)

- **[`/src`](./src)** - **Active Systems (现役系统)**
  - Contains maintained frameworks like Christianity, Buddhism, Islam, etc.
- **[`/legacy`](./legacy)** - **Deprecated Systems (已废弃系统)**
  - Ancient, read-only codebases like Greek Mythology, Ancient Egyptian, etc.
- **[`/RFCs`](./RFCs)** - **Protocols & Standards (协议与标准)**
  - Technical documentation on universal concepts (e.g., Karma, Sin, Reincarnation).

## 🏗️ Architecture (架构)

How these systems are built:

* **Core Kernel**: The fundamental dogma (e.g., Trinity, Emptiness).
    * *核心内核：基础教条（如：三位一体、空性）。*
* **Runtime Environment**: Rituals, meditation practices, daily routines.
    * *运行时环境：仪式、冥想实践、日常修行。*
* **User Interface**: Temples, vestments, iconography.
    * *用户界面：寺庙、法衣、圣像。*

---

## 🔀 Branching Strategy (分支策略)

How these systems evolve over time:

* **Main Branch**: Orthodox/Traditional interpretations.
    * *主分支：正统/传统解释。*
* **Major Forks**: e.g., Protestantism forked from Catholicism in `v15.17`.
    * *主要分叉：例如，新教在 v15.17 版本从天主教分叉（Fork）。*
* **Deprecated Features**: Animal sacrifice, indulgences (patched out).
    * *已废弃功能：动物献祭、赎罪券（已通过补丁移除）。*
